Pick the channel that matches your target platform.

| Channel | Coordinate | Platforms |
| --- | --- | --- |
| **pip (Python)** | `bithuman` 1.17.2 | macOS arm64, Linux x86_64, Linux aarch64, Windows x86_64 |
| **SwiftPM (Apple)** | `bitHumanKit` (`from: "0.8.1"`) | macOS arm64, iOS device, iOS simulator |
| **Maven Central (Android)** | `ai.bithuman:sdk:1.17.1` | Android `arm64-v8a`, minSdk 29 |
| **Homebrew (CLI)** | `bithuman-product/bithuman/bithuman` | macOS arm64 |
| **GitHub Releases (CLI)** | `bithuman` tarballs | Linux x86_64, Linux aarch64, Windows x86_64 |
| **Flutter pub** | `bithuman` (publish pending — install via git dep) | macOS · iOS · Android |

Every channel reads the same `.imx` model format and produces identical output. All third-party dependencies are bundled inside the SDK — your app needs only the dependency above.

## Python — `pip`

```bash
pip install bithuman --upgrade
```

- Python 3.10 – 3.13.
- Wheels: macOS arm64, Linux x86_64, Linux aarch64, Windows x86_64.

Smoke test:

```python
import asyncio, os
from bithuman import AsyncBithuman

async def main():
    runtime = await AsyncBithuman.create(
        model_path="avatar.imx",
        api_secret=os.environ["BITHUMAN_API_SECRET"],
    )
    print("frame size:", runtime.frame_width, "x", runtime.frame_height)
    await runtime.stop()

asyncio.run(main())
```

Full guide: [Python quickstart](/getting-started/quickstart).

## Swift — SwiftPM

In `Package.swift`:

```swift
dependencies: [
    .package(
        url: "https://github.com/bithuman-product/bithuman-sdk-public.git",
        from: "0.8.1"
    ),
],
targets: [
    .target(
        name: "MyApp",
        dependencies: [
            .product(name: "bitHumanKit", package: "bithuman-sdk-public"),
        ]
    )
]
```

In Xcode: **File → Add Package Dependencies…** → paste `https://github.com/bithuman-product/bithuman-sdk-public.git` → pick `0.8.1` (or the latest tag).

Smoke test:

```swift
import bitHumanKit

let runtime = try await Bithuman.createRuntime(modelPath: modelURL)
switch runtime {
case .essence(let essence):
    print("frame size:", essence.resolution)
case .expression:
    break
}
```

The SDK is distributed as a binary XCFramework (macOS arm64 + iOS device arm64 + iOS simulator arm64). Zero transitive SwiftPM dependencies.

Full guide: [Swift quickstart](/swift-sdk/quickstart).

## Kotlin / Android — Maven Central

In `app/build.gradle.kts`:

```kotlin
android {
    defaultConfig {
        ndk { abiFilters += setOf("arm64-v8a") }
        minSdk = 29
    }
}

dependencies {
    implementation("ai.bithuman:sdk:1.17.1")
}
```

Smoke test:

```kotlin
import ai.bithuman.sdk.Bithuman

val runtime = Bithuman.createRuntime("/path/to/avatar.imx")
println("frame size: ${runtime.frameWidth} x ${runtime.frameHeight}")
runtime.close()
```

AAR ships `arm64-v8a` only. Full guide: [Kotlin quickstart](/kotlin-sdk/quickstart).

## CLI — Homebrew

```sh
brew install bithuman-product/bithuman/bithuman
```

```sh
bithuman doctor          # verify host setup + API key
bithuman avatar          # local avatar in your browser at :8080
bithuman voice           # spoken conversation in the terminal
bithuman text            # text-only chat, stdin → stdout
```

`voice` and `text` auto-select cloud (OpenAI) when `OPENAI_API_KEY` is set, on-device otherwise. Explicit `--openai` / `--local` flags override the auto-pick.

No-Homebrew alternative (Linux, manual macOS install):

```sh
curl -fsSL https://github.com/bithuman-product/bithuman-sdk-public/releases/latest/download/install.sh | sh
```

Full guide: [CLI reference](/getting-started/cli).

## Flutter — `pub`

```yaml
# pubspec.yaml
dependencies:
  bithuman: ^1.0.0
```

```bash
flutter pub get
```

<Note>
Pub.dev publish is pending. Until then, install via git dep:

```yaml
dependencies:
  bithuman:
    git:
      url: https://github.com/bithuman-product/bithuman-apps.git
      path: flutter/bithuman
```
</Note>

The Flutter plugin gives you one Dart codebase covering macOS, iOS, and Android. Built-in echo cancellation per platform. Full guide: [Flutter plugin](/integrations/flutter).

## Authentication

Every channel reads the same credential. Platform convention determines the variable name:

```sh
export BITHUMAN_API_SECRET=...   # Python, Kotlin, REST API, CLI
export BITHUMAN_API_KEY=...      # Swift (Apple convention)
```

Sign in at [www.bithuman.ai → Developer → API Keys](https://www.bithuman.ai/#developer) to fetch a secret. Full auth flow: [Authentication](/getting-started/authentication).
