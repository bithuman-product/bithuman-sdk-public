bitHuman runs on every major platform. Inference happens on the device — only a one-request-per-minute billing heartbeat touches the network, and audio-only mode is fully offline.

That means:
- **Low latency** — frames render in under 200 ms end-to-end.
- **Privacy** — audio and video never leave the device.
- **No cloud dependency** — sessions continue through intermittent connectivity.
- **Predictable cost** — flat per-minute credit rate, no variable compute charges.

## Full matrix

| Platform | Models | SDKs | Minimum hardware |
|---|---|---|---|
| **iOS** | Essence | Swift, Flutter | iPhone 17 Pro+ |
| **iPadOS** | Essence + Expression | Swift, Flutter | iPad Pro M4+ |
| **macOS Apple Silicon** | Essence + Expression | Python, Swift, Flutter, CLI | Any Apple Silicon for Essence · M3+ for Expression |
| **macOS Intel** | Essence | Python | Any Intel Mac |
| **Android** | Essence | Kotlin, Flutter | `arm64-v8a`, Android 10+ |
| **Linux x86_64** | Essence | Python, CLI | Any modern CPU, 4 GB RAM |
| **Linux ARM64** | Essence | Python, CLI | Raspberry Pi 4B+, 4 GB RAM |
| **Windows x86_64** | Essence | Python, CLI | Any modern CPU, 4 GB RAM |
| **NVIDIA GPU (Docker)** | Expression | Docker container | 8 GB+ VRAM, Linux |
| **bitHuman Cloud** | Essence + Expression | REST API · LiveKit plugin | API key only |

Every platform delivers 25 FPS at the model's native resolution.

## Platform details

### iOS / iPadOS

Native Swift apps via [`bitHumanKit`](/swift-sdk/overview), or one Dart codebase via the [Flutter plugin](/integrations/flutter).

- **iPhone** runs Essence on iPhone 17 Pro+. Expression on iPhone isn't currently supported — use Essence.
- **iPad** runs both Essence and Expression on iPad Pro M4+.

Audio-only mode runs without an API key, fully offline.

**Examples:** [iOS guide](/swift-sdk/ios) · [Swift quickstart](/swift-sdk/quickstart) · [Essence on Swift](/swift-sdk/essence) · [Flutter plugin](/integrations/flutter).

### macOS

Three integration paths:

- **Python SDK** — `pip install bithuman` (Intel and Apple Silicon, Essence; Expression on M3+).
- **Swift SDK** — `bitHumanKit` for native Mac apps.
- **bitHuman CLI** — `brew install bithuman-product/bithuman/bithuman` (Apple Silicon).

```bash
pip install bithuman           # Essence + Expression on M3+
brew install bithuman-product/bithuman/bithuman
```

**Examples:** [Expression on macOS](/examples/apple-expression) · [Apple Local agent](/examples/apple-local) · [Halo Desktop App](/examples/halo-macos).

### Android

Native Kotlin via [`ai.bithuman:sdk`](/kotlin-sdk/overview), or Dart via the [Flutter plugin](/integrations/flutter). `arm64-v8a` only, Android 10+. Expression on Android isn't supported — use Essence.

**Examples:** [Kotlin quickstart](/kotlin-sdk/quickstart) · [Flutter plugin](/integrations/flutter).

### Linux x86_64

The default production target. Essence runs anywhere via the Python SDK or Rust CLI. Expression runs in a Docker container on any NVIDIA GPU with 8 GB+ VRAM.

```bash
pip install bithuman                                       # Essence on CPU
docker run --gpus all -p 8089:8089 \                       # Expression on GPU
    -e BITHUMAN_API_SECRET=$BITHUMAN_API_SECRET \
<!-- TODO(infra): migrate from `sgubithuman` (personal Docker Hub namespace) to `ghcr.io/bithuman-product/expression-avatar` or a properly-owned `bithuman` org Docker Hub namespace before this doc goes external. -->
    docker.io/sgubithuman/expression-avatar:latest
```

**Examples:** [Audio Clip](/examples/audio-clip) · [Self-hosted GPU](/deployment/self-hosted-gpu) · [AI Conversation](/examples/ai-conversation).

### Linux ARM64 (Raspberry Pi)

Essence runs natively on Raspberry Pi 4B+ via the Python SDK. Ideal for kiosks, museum displays, signage, and IoT — no idle timeout, no cloud dependency.

```bash
pip install bithuman
python kiosk_agent.py
```

**Example:** [Raspberry Pi deployment](/examples/raspberry-pi).

### Windows x86_64

Essence runs natively. For Expression on GPU, use WSL2 + the Docker container.

```bash
pip install bithuman
python your_agent.py
```

### NVIDIA GPU (Docker)

Expression runs on any NVIDIA GPU with 8 GB+ VRAM via an official Docker container. Inference runs on your hardware — audio and video never leave your network.

```bash
docker run --gpus all -p 8089:8089 \
    -e BITHUMAN_API_SECRET=$BITHUMAN_API_SECRET \
    docker.io/sgubithuman/expression-avatar:latest
```

Tested GPU tiers: RTX 4070 (8 GB), T4 (16 GB), A10 (24 GB), A100 (40/80 GB), H100 (80 GB).

**Example:** [Self-hosted GPU](/deployment/self-hosted-gpu).

### bitHuman Cloud

Managed runtime. Hit the [REST API](/api-reference/overview) or use the [LiveKit plugin](/deployment/livekit-cloud-plugin) — no local hardware, no model files to manage. This is the only path where inference doesn't run on your hardware.

```bash
pip install livekit-plugins-bithuman
```

## Choosing the right platform

| Your use case | Pick | Why |
|---|---|---|
| Fastest setup, no infra | bitHuman Cloud | Both models, API key only |
| Edge kiosk, 24/7 display | Linux ARM64 (Raspberry Pi) | Essence has no idle timeout |
| Native iPad app, custom portraits | Swift SDK + Expression | On-device dynamic faces |
| Native iPhone app | Swift SDK + Essence | Expression on iPhone not yet supported |
| Native Android app | Kotlin SDK or Flutter plugin | Essence on `arm64-v8a` |
| Cross-platform mobile/desktop | Flutter plugin | One Dart codebase, three platforms |
| Native macOS, kiosk / multi-instance | Swift or Python SDK + Essence | Lower memory, no idle timeout |
| Native macOS, dynamic faces | Swift SDK + Expression | M3+ on-device |
| On-prem, data sovereignty | Self-hosted GPU or CPU | No data leaves your network |
| High-concurrency self-hosted | Linux + Essence | Lower cost per stream |
| Custom-portrait avatars at scale | Self-hosted GPU + Expression | Multiple concurrent sessions per GPU |
| Windows desktop | Python SDK or Rust CLI | Native wheels, no WSL needed |
| IoT / embedded | Linux ARM64 + Essence | Runs on 4 GB RAM, 1–2 CPU cores |
