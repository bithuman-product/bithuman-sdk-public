## On-device, everywhere

bitHuman is designed for on-device local inference. Every supported platform runs the avatar model directly on local hardware — no cloud round-trip for rendering. Audio processing, lip-sync model execution, and frame generation all happen on the device in front of the user.

The only network call during an on-device session is a lightweight billing heartbeat (1 request per minute, under 1 KB). Audio-only mode on Apple Silicon requires zero network calls and works fully offline.

This means:
- **Lower latency** — no cloud round-trip; frames render in under 200 ms end-to-end
- **Full privacy** — audio and video never leave the device
- **No cloud dependency** — sessions run even when connectivity is intermittent or absent
- **Predictable cost** — no variable cloud compute charges; pay only the flat per-minute credit rate

## Platform details

### Linux x86_64 — servers and containers

The most common deployment target for production workloads. Essence runs on bare metal or in Docker with no GPU. Expression runs in the official Docker container on any NVIDIA GPU with 8 GB+ VRAM.

```bash
# Essence on CPU — any Linux server
pip install bithuman
python your_agent.py

# Expression on NVIDIA GPU — Docker container
docker run --gpus all -p 8089:8089 \
    -e BITHUMAN_API_SECRET=your_api_secret \
    docker.io/sgubithuman/expression-avatar:latest
```

All inference runs locally. No audio or video data is sent to bitHuman servers.

**Examples:** [Local Essence](/examples/audio-clip) | [Self-Hosted GPU](/examples/self-hosted-plugin) | [AI Conversation](/examples/ai-conversation)

### Linux ARM64 — Raspberry Pi and edge devices

Essence runs natively on ARM64 Linux, including Raspberry Pi 4B+ with 4 GB RAM. Ideal for edge kiosks, museum displays, digital signage, and IoT deployments where the device runs 24/7 with no cloud dependency.

```bash
# On a Raspberry Pi 4B+ running Raspberry Pi OS (64-bit)
pip install bithuman
# Download your .imx model, then run
python kiosk_agent.py
```

On-device local inference means the kiosk operates even when network connectivity is intermittent. The Essence model has no idle timeout, supporting continuous 24/7 operation.

**Example:** [Raspberry Pi deployment with systemd auto-start](/examples/raspberry-pi)

### macOS — Intel and Apple Silicon

Essence runs on any Mac via the Python SDK. Expression runs on-device on Apple Silicon M3+ via the Swift SDK (`bitHumanKit`) or the Python SDK (which delegates to the bundled Swift runtime automatically).

```bash
# Essence on any Mac
pip install bithuman

# Expression on Apple Silicon M3+ (Python SDK auto-delegates to Swift runtime)
pip install bithuman
python expression_demo.py

# Expression via Swift SDK
import bitHumanKit
```

The `bithuman-cli` Homebrew tool and the bitHuman Halo desktop app both run fully on-device on Apple Silicon — STT, LLM, TTS, and lip-synced avatar all execute locally.

**Examples:** [Expression on macOS](/examples/apple-expression) | [Apple Local](/examples/apple-local) | [Halo Desktop App](/examples/halo-macos)

### Windows x86_64

Essence runs natively on Windows via Python wheels for Python 3.9-3.14. No WSL required for Essence. For Expression on GPU, use WSL2 with the Docker container.

```bash
# Essence on Windows (native)
pip install bithuman
python your_agent.py
```

**Example:** Same Python examples as Linux — [Audio Clip](/examples/audio-clip) | [Microphone](/examples/microphone)

### iOS and iPadOS — Swift SDK

Expression runs on-device on iPhone 16 Pro+ (A18 Pro chip) and iPad Pro M4+ via the `bitHumanKit` Swift Package. All inference — speech recognition, LLM, text-to-speech, and lip-synced avatar rendering — runs on the device's GPU and Neural Engine.

```swift
import bitHumanKit

var config = VoiceChatConfig()
config.systemPrompt = "You are a helpful assistant."
let weights = try await ExpressionWeights.ensureAvailable()
config.avatar = AvatarConfig(modelPath: weights, portraitPath: portraitURL)
config.apiKey = ProcessInfo.processInfo.environment["BITHUMAN_API_KEY"]

let chat = VoiceChat(config: config)
try await chat.start()
```

Audio-only mode (no avatar) works without an API key and is fully offline and unmetered.

**Examples:** [iOS Avatar](/swift-sdk/ios) | [Swift SDK Quickstart](/swift-sdk/quickstart)

### NVIDIA GPU — Docker container

The Expression model runs on any NVIDIA GPU with 8 GB+ VRAM via an official Docker container. Supports up to 9 concurrent avatar sessions per GPU. All inference runs on your infrastructure — no audio or video data leaves your network.

```bash
docker run --gpus all -p 8089:8089 \
    -e BITHUMAN_API_SECRET=your_api_secret \
    docker.io/sgubithuman/expression-avatar:latest
```

Tested GPU tiers: RTX 4070 (8 GB), T4 (16 GB), A10 (24 GB), A100 (40/80 GB), H100 (80 GB).

**Example:** [Self-Hosted GPU deployment](/deployment/self-hosted-gpu)

### bitHuman Cloud — managed runtime

For teams that prefer managed infrastructure, bitHuman Cloud runs both Essence and Expression on bitHuman's servers. Use the LiveKit plugin or REST API from any platform — no local GPU or model files required.

```bash
pip install livekit-plugins-bithuman
```

This is the only surface where inference does not run on your hardware. All other surfaces perform on-device local inference.

**Example:** [Cloud Plugin](/deployment/livekit-cloud-plugin)

## Choosing the right platform

| Your use case | Recommended platform | Why |
|---|---|---|
| Web app, fastest setup | bitHuman Cloud | No infrastructure to manage |
| Edge kiosk, 24/7 display | Linux ARM64 (Raspberry Pi) or Linux x86_64 | On-device, no idle timeout, no cloud dependency |
| Native iOS/iPadOS app | Swift SDK on-device | Local inference, full privacy, app store distribution |
| Native macOS app | Swift SDK or Python SDK on-device | Local inference, Halo desktop app, bithuman-cli |
| On-prem enterprise, data sovereignty | Self-hosted GPU (Docker) or Self-hosted CPU | No data leaves your network |
| High-volume dynamic avatars | Self-hosted GPU (Docker) | Up to 9 sessions per GPU, any face image |
| Windows desktop app | Python SDK (Essence) | Native wheels, no WSL needed |
| IoT / embedded | Linux ARM64 (Essence) | Runs on 4 GB RAM, 1-2 CPU cores |
