## May 2026

### v1.17.x ŌĆö server-side avatar Realtime + Flutter rename (2026-05-14)

- **`bithuman avatar --openai`** ŌĆö new CLI mode that runs OpenAI Realtime on your workstation while the browser displays the avatar. Mic captures and speakers play locally; the avatar renders in any browser. Default Realtime model is `gpt-realtime-mini`; override with `--model gpt-realtime` for higher quality.
- **Auto-pick backend for `voice` / `text`.** Both subcommands now infer the backend at startup: cloud (OpenAI) when `OPENAI_API_KEY` is set, on-device (`--local`) otherwise. Explicit `--openai` / `--local` flags still win.
- **Interactive terminal UI for `voice`.** On a TTY, `bithuman voice` shows live mic and bot level meters plus a scrolling transcript. Set `BITHUMAN_NO_TUI=1` to fall back to plain stdout.
- **Flutter plugin renamed `bithuman_avatar` ŌåÆ `bithuman`.** One Dart codebase covers macOS, iOS, and Android, with built-in echo cancellation per platform. The settings sheet hot-updates the system prompt mid-session. Pub.dev publish is pending; install via git dep until then.
- **`BithumanRealtimeOpenAI` Swift library removed.** Its only consumer was the retired Swift `bithuman-cli`; the canonical OpenAI Realtime path is now `bithuman voice --openai` / `bithuman avatar --openai` via the Rust CLI.
- **Reference apps:** native macOS Expression app, iPadOS Expression app (revived this release with Picture-in-Picture), and cross-platform Essence Flutter example. iPhone Expression remains parked while the per-app memory budget firms up ŌĆö use Essence on iPhone.

### v1.16.0 ŌĆö streaming API on Swift and Kotlin (2026-05-14)

Streaming-first refactor for the Swift and Kotlin SDKs. Long sessions stay smooth ŌĆö per-tick cost is now flat regardless of how much audio has accumulated.

- **Swift** ŌĆö `pushAudio` / `frames()` / `resetStream()` on `EssenceRuntime`.
- **Kotlin** ŌĆö `pushAudio` / `pullFrame` / `ticksAvailable` / `resetStream()` on `ai.bithuman.sdk.Runtime`. Maven `ai.bithuman:sdk:1.16.0+`.
- **CLI** ŌĆö default OpenAI Realtime model is `gpt-realtime-mini` (cost-efficient tier).

### v1.12.0 ŌĆö unified SDK release (2026-05-12)

First unified release across all SDKs from a single source. Install matrix:

- `pip install bithuman` ŌåÆ **1.12.0** (macOS arm64, Linux x86_64, Linux aarch64 wheels)
- SwiftPM `bitHumanKit` from `bithuman-sdk-public`
- Maven `ai.bithuman:sdk:1.12.0` ŌĆö first publish on Maven Central, `arm64-v8a`, minSdk 29
- `brew install bithuman-product/bithuman/bithuman`

Highlights:

- **Same runtime, four SDKs.** Python, Swift, Kotlin, and the Rust CLI all produce identical output.
- **Maven Central, first publish.** `ai.bithuman:sdk:1.12.0` AAR with bundled native dependencies.
- **Linux + Windows wheels.** Native wheels for all supported Python versions, no WSL needed.
