bitHuman ships from one source tag (`libessence-v1.17.0`) into multiple package
registries. Pick the channel that matches your target platform.

| Channel | Coord | Platforms |
| ------- | ----- | --------- |
| pip (Python) | `bithuman` 1.12.4 | macOS arm64, Linux x86_64, Linux aarch64 |
| SwiftPM (Apple) | `Bithuman` (`from: "1.17.0"`) | macOS arm64, iOS device, iOS simulator |
| Maven Central (Android) | `ai.bithuman:sdk:1.17.0` | Android `arm64-v8a`, minSdk 29 |
| Homebrew (CLI) | `bithuman` 1.17.0 | macOS arm64 |
| GitHub Releases (CLI) | `bithuman` 1.17.0 tarballs | Linux x86_64, Linux aarch64, Windows x86_64 |
| Flutter pub | `bithuman` (renamed from `bithuman_avatar`) | macOS · iOS · Android — pub.dev publish queued |

<Note>
The Python wheel is still on the legacy 1.12.4 batched API while the
streaming wheel is queued behind the unified Rust + PyO3 build. Swift, Kotlin,
the Rust crate, and the engine's C ABI expose the [streaming push/pull API](/getting-started/architecture#streaming-api-c-abi-v6)
introduced in v1.16.0.
</Note>

For the underlying architecture (Layer 1/2/3, the role of `libessence`),
see [Architecture](/getting-started/architecture).

## Python — `pip`

```bash
pip install bithuman --upgrade   # 1.12.4 (legacy batched API)
```

- Python 3.10 – 3.13 (CPython).
- Wheels: macOS arm64, Linux x86_64 (`manylinux_2_28`), Linux aarch64 (`manylinux_2_28`).
- Windows wheels are tracked for a follow-up release.

Smoke-test in one line:

```python
from bithuman import Avatar
with Avatar.load("model.imx") as avatar:
    for frame in avatar.compose("speech.wav"):
        # frame.bgr is (H, W, 3) uint8 BGR
        ...
```

Full guide: [Python quickstart](/getting-started/quickstart).

## Swift — SwiftPM

In `Package.swift`:

```swift
dependencies: [
    .package(
        url: "https://github.com/bithuman-product/bithuman-sdk",
        from: "1.17.0"
    ),
],
targets: [
    .target(
        name: "MyApp",
        dependencies: [
            .product(name: "Bithuman", package: "bithuman-sdk"),
        ]
    )
]
```

In Xcode: **File → Add Package Dependencies…** → paste the repo URL → pick
`1.17.0` (or the latest tagged version).

```swift
import Bithuman

let avatar = try Avatar(modelPath: modelURL)
try avatar.compose(audioFile: speechURL) { frame in
    if let cg = frame.cgImage() { /* hand to SwiftUI / UIKit / AppKit */ }
}
```

The package ships a 3-slice xcframework (macOS arm64 + iOS arm64 device + iOS
arm64 simulator). Static link of `libessence` — zero transitive SwiftPM
dependencies. Full guide: [Swift quickstart](/swift-sdk/quickstart).

## Kotlin / Android — Maven Central

In `app/build.gradle.kts`:

```kotlin
android {
    defaultConfig {
        ndk { abiFilters += setOf("arm64-v8a") }
        minSdk = 29
    }
}

dependencies {
    implementation("ai.bithuman:sdk:1.17.0")
}
```

```kotlin
import ai.bithuman.sdk.Avatar

Avatar.load("/path/to/model.imx").use { avatar ->
    avatar.composeFromFile("/path/to/speech.wav").forEach { frame ->
        // frame.bgr is width*height*3 packed BGR uint8
    }
}
```

AAR is ~14 MB. `arm64-v8a` only (other ABIs deferred). Full guide:
[Kotlin quickstart](/kotlin-sdk/quickstart).

## CLI — Homebrew

```sh
brew tap bithuman-product/bithuman
brew install bithuman
```

The canonical formula is `bithuman`. `bithuman-cli` is kept as a tap alias
for back-compat — existing scripts using `brew install bithuman-cli` continue
to work unchanged.

```sh
bithuman doctor          # verify host setup + API key presence
bithuman avatar          # browser-served local avatar on :8080 (default)
bithuman avatar --openai # server-side OpenAI Realtime + browser avatar (v1.17.0)
bithuman voice           # auto-picks cloud (OPENAI_API_KEY) or --local
bithuman text            # same auto-pick rule, non-interactive stdin → stdout
```

`voice` and `text` infer the backend at startup: cloud (OpenAI Realtime / Chat Completions) when `OPENAI_API_KEY` is set, on-device (`--local`) otherwise. Explicit `--openai` / `--local` flags still win. On a TTY, `voice` renders a ratatui status pill + 🎙 mic / 🔊 bot level bars + scrolling transcript; set `BITHUMAN_NO_TUI=1` to fall back to plain stdout.

No-Homebrew alternative (curl-pipe):

```sh
curl -fsSL https://github.com/bithuman-product/bithuman-sdk/releases/latest/download/install.sh | sh
```

You'll still need the runtime dylibs the CLI links against:

```sh
brew install onnxruntime hdf5 jpeg-turbo webp ffmpeg
```

Full guide: [CLI reference](/getting-started/cli).

## Flutter — `pub`

The cross-platform Flutter plugin (renamed from `bithuman_avatar` →
`bithuman` in v1.17.0) wraps the Swift and Kotlin bindings under one
Dart codebase covering macOS + iOS + Android.

```yaml
# pubspec.yaml
dependencies:
  bithuman: ^1.17.0
```

```bash
flutter pub get
```

<Note>
Pub.dev publish is queued. Until then, install via path / git dep:

```yaml
dependencies:
  bithuman:
    git:
      url: https://github.com/bithuman-product/bithuman-apps.git
      path: flutter/bithuman
```
</Note>

The plugin auto-selects the right Realtime transport per platform:
macOS + iOS use a WebSocket Realtime through the plugin's VP-IO
`RealtimeAudioIO` (Apple hardware AEC); Android uses flutter_webrtc's
libwebrtc native pipeline (libwebrtc native AEC). Single
`AvatarScreen` Dart UI; live prompt updates propagate to the running
session via `session.update`.

Full guide: [Flutter plugin](/integrations/flutter).

## Authentication

Every channel reads the same credential under one of two equivalent env-var
names (platform convention determines which):

```sh
export BITHUMAN_API_SECRET=...   # Python, Kotlin, REST API, CLI
export BITHUMAN_API_KEY=...      # Swift (Apple-platform convention)
```

Sign in at [www.bithuman.ai → Developer → API Keys](https://www.bithuman.ai/#developer)
to fetch a secret. Full auth flow: [Authentication](/getting-started/authentication).
