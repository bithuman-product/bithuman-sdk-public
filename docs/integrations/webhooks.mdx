bitHuman sends webhooks when key avatar events happen on your account — for example, when a user joins a room or a chat message is pushed.

## Quick setup

<Steps>
  <Step title="Open webhook settings">
    Go to [Developer → Webhooks](https://www.bithuman.ai/#developer) and enable the webhook toggle.

    <img src="/images/webhook-dashboard.jpg" alt="Webhook Configuration" style={{ borderRadius: '8px', width: '100%' }} />
  </Step>
  <Step title="Enter endpoint URL">
    Your webhook receiver. Must be HTTPS.
  </Step>
  <Step title="Select events">
    Choose **room.join**, **chat.push**, or both.
  </Step>
  <Step title="Reveal your signing secret">
    Click **Reveal signing secret** on the Webhooks settings panel. This is the value you'll HMAC-verify every incoming request against. Treat it like a password — store it in your secret manager, never check it into source control.
  </Step>
</Steps>

<Note>
The signing secret is the only authentication you need. Don't add a separate `Authorization: Bearer` header — incoming requests are authenticated by the HMAC signature alone.
</Note>

## What's in the payload

All payloads share the same envelope:

| Field | Type | Description |
|---|---|---|
| `event_id` | string | Unique per delivery. Use for idempotency. |
| `agent_code` | string | The agent that triggered the event. |
| `event_type` | string | `room.join` or `chat.push`. |
| `data` | object | Event-specific data. |
| `timestamp` | float | Unix timestamp (seconds, with milliseconds). |

### `room.join`

```json
{
  "event_id": "evt_01HM9XR3VK...",
  "agent_code": "A91XMB7113",
  "event_type": "room.join",
  "data": {
    "room_name": "customer-support",
    "participant_count": 1,
    "session_id": "session_xyz789"
  },
  "timestamp": 1705312200.123
}
```

### `chat.push`

```json
{
  "event_id": "evt_01HM9XR3WC...",
  "agent_code": "A91XMB7113",
  "event_type": "chat.push",
  "data": {
    "role": "user",
    "message": "Hello, I need help with my order",
    "session_id": "session_xyz789"
  },
  "timestamp": 1705312285.456
}
```

## Verify the signature

Every webhook request includes an `X-bitHuman-Signature` header of the form `sha256=<hex digest>`. Verify it as `HMAC-SHA256(signing_secret, raw_request_body)`.

<Warning>
HMAC the **raw request body bytes**, not a re-serialized JSON object. Any reformatting (whitespace, key reordering) breaks the signature.
</Warning>

<CodeGroup>

```python Flask (Python)
from flask import Flask, request, jsonify
import hmac, hashlib, os

app = Flask(__name__)
WEBHOOK_SECRET = os.environ["BITHUMAN_WEBHOOK_SECRET"].encode()
processed_event_ids: set[str] = set()

@app.post("/webhook")
def handle_webhook():
    raw = request.get_data()                           # raw bytes, not request.json
    signature = request.headers.get("X-bitHuman-Signature", "")
    if not verify(raw, signature):
        return jsonify({"error": "invalid signature"}), 401

    event = request.get_json()
    event_id = event.get("event_id")
    if event_id in processed_event_ids:                # idempotency
        return jsonify({"status": "duplicate"}), 200
    processed_event_ids.add(event_id)

    if event["event_type"] == "room.join":
        print(f"join: {event['data']['session_id']}")
    elif event["event_type"] == "chat.push":
        print(f"[{event['data']['role']}] {event['data']['message']}")
    return jsonify({"status": "ok"}), 200

def verify(raw: bytes, signature: str) -> bool:
    expected = hmac.new(WEBHOOK_SECRET, raw, hashlib.sha256).hexdigest()
    return hmac.compare_digest(f"sha256={expected}", signature)
```

```javascript Express (Node.js)
const express = require('express');
const crypto = require('crypto');

const app = express();
const WEBHOOK_SECRET = process.env.BITHUMAN_WEBHOOK_SECRET;
const processed = new Set();

// CRITICAL: capture the raw body — HMAC must hash bytes, not re-serialized JSON.
app.post('/webhook', express.raw({ type: 'application/json' }), (req, res) => {
  const signature = req.headers['x-bithuman-signature'] || '';
  if (!verify(req.body, signature)) {
    return res.status(401).json({ error: 'invalid signature' });
  }

  const event = JSON.parse(req.body.toString('utf8'));
  if (processed.has(event.event_id)) {                  // idempotency
    return res.json({ status: 'duplicate' });
  }
  processed.add(event.event_id);

  if (event.event_type === 'room.join') {
    console.log(`join: ${event.data.session_id}`);
  } else if (event.event_type === 'chat.push') {
    console.log(`[${event.data.role}] ${event.data.message}`);
  }
  res.json({ status: 'ok' });
});

function verify(rawBuffer, signature) {
  const expected = crypto
    .createHmac('sha256', WEBHOOK_SECRET)
    .update(rawBuffer)                                  // raw Buffer, not stringify
    .digest('hex');
  const expectedHeader = Buffer.from(`sha256=${expected}`);
  const givenHeader = Buffer.from(signature);
  return expectedHeader.length === givenHeader.length
    && crypto.timingSafeEqual(expectedHeader, givenHeader);
}

app.listen(3000, () => console.log('Listening on port 3000'));
```

</CodeGroup>

## Handle duplicate deliveries

Every delivery carries a unique `event_id`. Network blips and your own 5xx responses can cause the same event to be delivered more than once — always dedupe on `event_id` before processing.

## When (and how often) we retry

Failed deliveries (non-2xx responses or timeouts) retry automatically:

| Attempt | Delay |
|---|---|
| 1st retry | 1 second |
| 2nd retry | 5 seconds |
| 3rd retry | 30 seconds |

Maximum 3 retries. Your endpoint must respond within 30 seconds. Return 200 immediately and process work asynchronously if your handler is slow.

## Troubleshooting

| Symptom | Fix |
|---|---|
| 401 invalid signature | You're HMACing the wrong bytes. Verify against the raw request body, not a re-serialized JSON object. |
| Same event arrives twice | Expected. Dedupe on `event_id`. |
| Timeouts | Return 200 immediately and process asynchronously. |
| 404 not found | Double-check the endpoint URL on the [Developer → Webhooks](https://www.bithuman.ai/#developer) panel. |
| SSL error | Your endpoint needs a valid HTTPS certificate. |
