## 2. Install

Pick your channel. Swift, Kotlin, and the CLI ship from the `libessence-v1.16.0` source tag with the new streaming push/pull API; the Python wheel is still on `libessence-v1.12.4` (legacy batched API) while the `py-v1.16.0` publish is queued.

<Tabs>
  <Tab title="pip (Python)">
    ```bash
    pip install bithuman --upgrade   # 1.12.4 (legacy batched API)
    ```
    macOS arm64, Linux x86_64, Linux aarch64. Python 3.10 – 3.13.
  </Tab>
  <Tab title="SwiftPM (Apple)">
    ```swift
    .package(
        url: "https://github.com/bithuman-product/bithuman-sdk",
        from: "libessence-v1.16.0"
    )
    ```
    macOS arm64 + iOS device + iOS simulator slices in one xcframework.
  </Tab>
  <Tab title="Maven (Android)">
    ```kotlin
    dependencies {
        implementation("ai.bithuman:sdk:1.16.0")
    }
    ```
    `arm64-v8a`, minSdk 29. AAR ships from Maven Central.
  </Tab>
  <Tab title="brew (CLI)">
    ```bash
    brew install bithuman-product/bithuman/bithuman   # 1.16.0
    ```
    Cross-platform CLI binary. See [the CLI page](/swift-sdk/cli).
  </Tab>
</Tabs>

<Note>
The Python SDK includes `opencv-python-headless` automatically. Do **not** install `opencv-python` (full) separately — it conflicts with PyAV and causes FFmpeg warnings on macOS.
</Note>

<Tip>
**Source, changelog, and issue tracker** for the `bithuman` PyPI package live at [bithuman-sdk-public/python](https://github.com/bithuman-product/bithuman-sdk-public/tree/main/python). The runtime source is private (signing material is baked in), but the public repo mirrors the README + changelog and is where to file bugs or feature requests.

## 3. Run Your First Avatar

You need a `.wav` audio file to drive the avatar. A sample `speech.wav` is included in each
[example directory](https://github.com/bithuman-product/bithuman-sdk-public/tree/main/Examples), or generate your own with any TTS service.

### Option A — CLI (fastest, no coding)

```bash
export BITHUMAN_API_SECRET=your_api_secret
bithuman generate avatar.imx --audio speech.wav --output demo.mp4
```

Open `demo.mp4` to see your avatar talking.

<Note>
  Don't have a WAV yet? Grab the bundled sample in one line:
  `curl -O https://raw.githubusercontent.com/bithuman-product/bithuman-sdk-public/main/Examples/python/local-essence/speech.wav`
</Note>

### Option B — Python

```python
import asyncio, os
from bithuman import AsyncBithuman
from bithuman.audio import load_audio, float32_to_int16

async def main():
    runtime = await AsyncBithuman.create(
        model_path="avatar.imx",
        api_secret=os.environ["BITHUMAN_API_SECRET"],
    )
    await runtime.start()

    pcm, sr = load_audio("speech.wav")
    pcm = float32_to_int16(pcm)
    chunk = sr // 25                          # one chunk per video frame
    for i in range(0, len(pcm), chunk):
        await runtime.push_audio(pcm[i : i + chunk].tobytes(), sr)
    await runtime.flush()

    try:
        async for frame in runtime.run():
            if frame.has_image:
                image = frame.bgr_image        # numpy (H, W, 3) uint8
            if frame.end_of_speech:
                break
    finally:
        await runtime.stop()

asyncio.run(main())
```

Want to display the frames live? See [Audio Clip](/examples/audio-clip) for the OpenCV + speaker playback pattern.

[Full working example on GitHub](https://github.com/bithuman-product/bithuman-sdk-public/tree/main/Examples/python/local-essence)

## Key Concepts

| Concept | Description |
|---------|-------------|
| **Runtime** | `AsyncBithuman` instance that processes audio into video |
| **push_audio** | Feed audio bytes — avatar lip-syncs in real-time |
| **flush** | Signals end of audio input |
| **run()** | Async generator that yields frames at 25 FPS |
| **Frame** | Contains `.bgr_image` (numpy), `.audio_chunk`, `.end_of_speech` |

## Troubleshooting

<AccordionGroup>
  <Accordion title="ModuleNotFoundError: No module named 'bithuman'">
    The SDK is not installed. Run:
    ```bash
    pip install bithuman --upgrade
    ```
    Make sure you're using the correct Python environment (virtualenv, conda, etc.).
  </Accordion>

  <Accordion title="Authentication failed / 401 error">
    Your API secret is invalid or missing. Check:
    1. You copied the full secret from [Developer → API Keys](https://www.bithuman.ai/#developer)
    2. The `api_secret` parameter or `BITHUMAN_API_SECRET` env var is set correctly
    3. Your account is active with available credits

    Quick test:
    ```bash
    curl -X POST https://api.bithuman.ai/v1/validate \
      -H "api-secret: YOUR_SECRET"
    ```
  </Accordion>

  <Accordion title="Avatar shows but no lip movement">
    The avatar needs audio input to animate:
    1. Ensure you're calling `push_audio()` with valid audio data
    2. Call `flush()` after pushing all audio
    3. Check that the audio is 16-bit PCM format (use `float32_to_int16()` helper)
    4. Verify audio sample rate matches the file (typically 16000 or 44100)
  </Accordion>

  <Accordion title="Slow first startup (30-60 seconds)">
    The very first run is slow for two reasons:
    1. **Legacy format conversion** — older `.imx` files (v1 format) are automatically converted to the optimized v2 format. This takes ~30 seconds and only happens once per model file.
    2. **Model loading** — the first time a model is loaded into memory takes ~5-10 seconds.

    Subsequent runs in the same process start in under 2 seconds. To reduce perceived latency in production, keep the runtime alive between sessions instead of recreating it.
  </Accordion>

  <Accordion title="FileNotFoundError: avatar.imx not found">
    The model file path is wrong. Check:
    1. The `.imx` file exists at the path you specified
    2. Use an absolute path if running from a different directory
    3. Download a model from the [Explore](https://www.bithuman.ai/#explore) page if you don't have one
  </Accordion>
</AccordionGroup>
