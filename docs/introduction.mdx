bitHuman puts a real-time, lip-synced talking avatar in your app. Send
audio in, get animated video out at 25 FPS with sub-200 ms latency. The
rendering runs **on the device** — no cloud round-trip, no PII leaving
the box, no network dependency for the avatar itself.

One engine ([`libessence`](/getting-started/architecture)) drives every
surface: the CLI, the Python / Swift / Kotlin / Flutter SDKs, and the
cloud REST API. They all read the same `.imx` avatar file and produce
identical frames — pick the surface that matches what you're building;
the mental model never changes.

## Get started

Choose your surface. The CLI is the fastest way to see it work — no
code, no language toolchain.

<Tabs>
  <Tab title="CLI (no code)">
    The universal one-liner — macOS and Linux:

    ```bash
    curl -fsSL https://github.com/bithuman-product/bithuman-sdk-public/releases/latest/download/install.sh | sh
    ```

    macOS via Homebrew (auto-pulls native deps):

    ```bash
    brew install bithuman-product/bithuman/bithuman
    ```

    Then:

    ```bash
    bithuman doctor      # verify host + API key
    bithuman avatar      # opens http://127.0.0.1:8080 with the avatar
    bithuman voice       # spoken conversation in your terminal
    bithuman text        # text-only chat, stdin → stdout
    ```

    `voice` / `text` auto-pick cloud (when `OPENAI_API_KEY` is set) or
    fully on-device. [CLI reference →](/getting-started/cli)
  </Tab>

  <Tab title="Python">
    ```bash
    pip install bithuman --upgrade
    ```

    Python 3.10–3.13. Wheels for macOS arm64, Linux x86_64 / aarch64,
    Windows x86_64. The runtime + all native deps are in the wheel.

    ```python
    import asyncio, os
    from bithuman import AsyncBithuman

    async def main():
        rt = await AsyncBithuman.create(
            model_path="avatar.imx",
            api_secret=os.environ["BITHUMAN_API_SECRET"],
        )
        print(rt.frame_width, "x", rt.frame_height)
        await rt.stop()

    asyncio.run(main())
    ```

    [Python SDK →](/sdks/python)
  </Tab>

  <Tab title="Swift">
    In Xcode: **File → Add Package Dependencies…** →
    `https://github.com/bithuman-product/bithuman-sdk-public.git` →
    pick the latest tag.

    ```swift
    // .product(name: "bitHumanKit", package: "bithuman-sdk-public")
    import bitHumanKit

    let runtime = try await Bithuman.createRuntime(modelPath: modelURL)
    ```

    Binary XCFramework, zero transitive SwiftPM deps. macOS arm64 +
    iOS device + simulator. [Swift SDK →](/sdks/swift)
  </Tab>

  <Tab title="Kotlin">
    ```kotlin
    // app/build.gradle.kts
    android { defaultConfig {
        ndk { abiFilters += setOf("arm64-v8a") }
        minSdk = 29
    } }
    dependencies { implementation("ai.bithuman:sdk:1.17.3") }
    ```

    ```kotlin
    import ai.bithuman.sdk.Bithuman
    val runtime = Bithuman.createRuntime("/path/to/avatar.imx")
    ```

    AAR ships `arm64-v8a`, Android 10+. [Kotlin SDK →](/sdks/kotlin)
  </Tab>

  <Tab title="Flutter">
    ```yaml
    # pubspec.yaml
    dependencies:
      bithuman: ^1.0.0
    ```

    One Dart codebase across macOS, iOS, and Android, with built-in
    per-platform echo cancellation.

    <Note>

    [Flutter plugin →](/integrations/flutter)
  </Tab>

  <Tab title="REST API">
    No install — call `api.bithuman.ai` from any language.

    ```bash
    # Make a hosted agent speak (create AGENT_CODE at bithuman.ai).
    curl -X POST https://api.bithuman.ai/v1/agent/AGENT_CODE/speak \
      -H "api-secret: $BITHUMAN_API_SECRET" \
      -H "content-type: application/json" \
      -d '{"message": "Hello from the REST API."}'
    ```

    [API reference →](/api-reference/overview)
  </Tab>
</Tabs>

## What you can do

<AccordionGroup>
  <Accordion title="Render a talking-head video from a script" icon="film">
    One command, no code:

    ```bash
    bithuman generate avatar.imx --audio speech.wav --output demo.mp4
    ```

    Batch-render at scale by looping over scripts. The same path is
    available in every SDK via the streaming API.
  </Accordion>

  <Accordion title="Have a spoken conversation" icon="microphone">
    ```bash
    export OPENAI_API_KEY=sk-...   # cloud LLM/voice
    bithuman voice                 # or `bithuman voice --local` for fully offline
    ```

    The avatar lip-syncs the bot's reply in real time. Half-duplex mic
    gating keeps it from talking over itself.
  </Accordion>

  <Accordion title="Embed an avatar in your app" icon="mobile">
    Push 16 kHz PCM as it arrives (mic, TTS, WebRTC); drain frames at
    25 FPS and hand them to your renderer. Same shape in Python, Swift,
    Kotlin, Flutter. See the [quickstart](/getting-started/quickstart).
  </Accordion>

  <Accordion title="Run a cloud voice agent with an avatar" icon="cloud">
    ```bash
    pip install livekit-plugins-bithuman
    ```

    Drop `bithuman.AvatarSession(...)` into a LiveKit agent worker —
    managed runtime, ~5 min setup. [Deployment →](/guides/deployment)
  </Accordion>

  <Accordion title="Self-host Expression on your own GPU" icon="server">
    ```bash
    docker run --gpus all -p 8089:8089 \
      -e BITHUMAN_API_SECRET=$BITHUMAN_API_SECRET \
      bithuman/expression-avatar:latest
    ```

    Point a LiveKit agent at `http://localhost:8089/launch`.
    [Deployment →](/guides/deployment)
  </Accordion>

  <Accordion title="Drop a chat avatar onto a website" icon="comments">
    A floating text / voice / video avatar widget, one script tag.
    [Deployment → embed →](/guides/deployment)
  </Accordion>
</AccordionGroup>

## Two avatar models

| Model | What it is | Best for |
|---|---|---|
| **[Essence](/getting-started/models#essence)** | A pre-built avatar identity in an `.imx`. Low memory, runs on every supported platform. **The default.** | Kiosks, mobile, edge, high-concurrency servers |
| **[Expression](/getting-started/models#expression)** | Animates *any* portrait at runtime. Higher close-up quality; needs more compute. | Native desktop apps, per-session custom faces, GPU cloud |

Both speak the same `.imx` format and the same SDK methods — swap them
without rewriting your integration.
