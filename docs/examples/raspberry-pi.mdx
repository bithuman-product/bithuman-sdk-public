The Essence avatar runs on a Raspberry Pi 4B via the same Python SDK that runs on any Linux server. This page walks through hardware, setup, an auto-start systemd unit, and the optimizations that matter for sustained 24/7 operation.

<Note>
Expression isn't supported on Raspberry Pi — CPU-only diffusion exceeds the real-time budget by an order of magnitude. Use **Essence** here.
</Note>

## Quick start

<Steps>
  <Step title="Hardware">
    - Raspberry Pi 4B (8 GB RAM recommended)
    - microSD card (32 GB+, Class 10) — or USB SSD for faster model loads
    - USB microphone
    - Stable internet connection (ethernet preferred)
    - **Separate computer** to host the web UI (recommended; running UI on the Pi alongside the agent causes slowdown)
  </Step>
  <Step title="Install OS">
    Use **Raspberry Pi OS (64-bit)** via Raspberry Pi Imager.
  </Step>
  <Step title="Set up Python">
    ```bash
    sudo apt update && sudo apt upgrade -y
    sudo apt install python3.11 python3.11-venv portaudio19-dev -y
    python3.11 -m venv bithuman-env
    source bithuman-env/bin/activate
    ```
  </Step>
  <Step title="Install bitHuman">
    ```bash
    pip install bithuman --upgrade livekit-agents openai
    ```
  </Step>
  <Step title="Configure">
    ```bash
    export BITHUMAN_API_SECRET="your_secret"
    export BITHUMAN_MODEL_PATH="/home/pi/avatar.imx"
    export OPENAI_API_KEY="your_openai_key"
    # If using LiveKit:
    export LIVEKIT_API_KEY="your_livekit_key"
    export LIVEKIT_API_SECRET="your_livekit_secret"
    export LIVEKIT_URL="wss://your-project.livekit.cloud"
    ```
  </Step>
  <Step title="Run">
    ```bash
    git clone https://github.com/bithuman-product/bithuman-sdk-public.git
    cd bithuman-sdk-public/Examples/python/local-essence
    python quickstart.py --model "$BITHUMAN_MODEL_PATH" --audio-file speech.wav
    ```
    For a LiveKit-driven agent, see the same directory's `agent.py`.
  </Step>
</Steps>

## Auto-start with systemd

Make the agent run automatically on boot:

```ini
# /etc/systemd/system/bithuman-agent.service
[Unit]
Description=bitHuman Avatar Agent
After=network.target

[Service]
Type=simple
User=pi
WorkingDirectory=/home/pi/bithuman-sdk-public/Examples/python/local-essence
EnvironmentFile=/home/pi/.bithuman.env
ExecStart=/home/pi/bithuman-env/bin/python agent.py start
Restart=always
RestartSec=10

[Install]
WantedBy=multi-user.target
```

Where `/home/pi/.bithuman.env` is a `chmod 600` file with your secrets:

```bash
BITHUMAN_API_SECRET=...
BITHUMAN_MODEL_PATH=/home/pi/avatar.imx
OPENAI_API_KEY=...
```

Enable and start:

```bash
sudo systemctl daemon-reload
sudo systemctl enable bithuman-agent
sudo systemctl start bithuman-agent
sudo systemctl status bithuman-agent
journalctl -u bithuman-agent -f      # follow logs
```

## Performance tips

```bash
# Pin CPU governor to performance.
echo 'performance' | sudo tee /sys/devices/system/cpu/cpu*/cpufreq/scaling_governor

# Trim background services if you don't need them.
sudo systemctl disable bluetooth
# sudo systemctl disable wifi        # only if you're on ethernet
```

- Add a swap file: `sudo dphys-swapfile swapon`.
- Store the `.imx` on a USB SSD — model load is the slowest step on a microSD.
- Monitor with `htop` and `vcgencmd measure_temp`; throttle at ~80°C affects sustained FPS.

## Common issues

| Symptom | Fix |
|---|---|
| Out of memory | Use the 8 GB Pi 4B and enable swap. |
| Slow performance | Switch to ethernet; check CPU temperature. |
| No audio input | `arecord -l` to list devices; `arecord -d 5 test.wav` to verify the mic. |
| Slow first-run load | First load converts `.imx` and warms the runtime. Subsequent loads are fast. |
