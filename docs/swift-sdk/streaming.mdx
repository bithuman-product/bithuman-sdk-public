The streaming push/pull API on `EssenceRuntime` is the right shape for conversations that run longer than a few seconds, or for any integration where audio arrives incrementally (microphone, TTS stream, WebRTC track).

Each push is constant cost; each pull returns one composed frame. Session length doesn't degrade per-tick performance.

## API surface

```swift
import bitHumanKit

// Create the runtime from an .imx file.
let runtime = try await Bithuman.createRuntime(modelPath: imxURL)
guard case .essence(let essence) = runtime else { return }

// Append PCM as it arrives. 16 kHz mono Int16 (or Float32).
try await essence.pushAudio(pcm)

// Drain composed CGImage frames.
for await frame in essence.frames() {
    if let frame { renderer.present(frame) }
}

// End-of-turn / barge-in: drop the audio cache so the next turn starts fresh.
try await essence.resetStream()
```

`pushAudio` and `frames()` are independent — push as audio arrives, drain on your display tick. Back-pressure is your choice: drain everything from the async stream as soon as it's available, or pace the read against a display link.

## Minimal driver

```swift
import bitHumanKit

func driveAvatar(imxURL: URL, audioChunks: AsyncStream<Data>) async throws {
    let runtime = try await Bithuman.createRuntime(modelPath: imxURL)
    guard case .essence(let essence) = runtime else { return }

    // Producer: push audio as it arrives.
    Task {
        for await pcm in audioChunks {
            try? await essence.pushAudio(pcm)
        }
    }

    // Consumer: render frames.
    for await frame in essence.frames() {
        if let frame {
            await MainActor.run { renderer.present(frame) }
        }
    }
}
```

## Fitting it into a SwiftUI app

Inside your existing voice-chat lifecycle, wire audio in and frames out:

```swift
// On WebRTC remote-audio sink (or AVAudioEngine tap):
try await essence.pushAudio(pcmChunk)

// Frames stream from a Task started at runtime creation:
Task {
    for await frame in essence.frames() {
        if let frame {
            await MainActor.run { renderer.present(frame) }
        }
    }
}
```

When there's no audio in flight, the runtime emits `nil` frames — keep displaying your last good frame, or render a static idle composite.

## When to call `resetStream()`

Reset at boundaries between turns: barge-in, end-of-utterance detection, agent switch, or any time the cached audio history is no longer relevant.
