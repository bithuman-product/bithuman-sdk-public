Get a working on-device voice agent running in your Mac, iPad, or iPhone app in under 10 minutes. All inference runs locally — the only network calls are the first-launch weights download and the metered billing heartbeat for avatar mode.

<Note>
**Two API surfaces** — both ship in the same `bitHumanKit` package:

- **`VoiceChat`** (this page) — the high-level voice agent. Built-in speech recognition, language model, and TTS. The fastest way to ship a talking on-device assistant.
- **`Bithuman.createRuntime` + `EssenceRuntime.frames()`** — the low-level streaming API. Push your own PCM, pull `CGImage` frames. Use this when you bring your own audio source (WebRTC, custom TTS, etc.). See [Streaming API](/swift-sdk/streaming) and the [iOS hello-world example](/examples/swift-ios-hello).
</Note>

## Prerequisites

- **Xcode 26+** — older Xcodes don't recognize some Swift 6 concurrency syntax `bitHumanKit` uses.
- **Minimum deployment targets**: macOS 26, iOS 26, iPadOS 26.
- **Apple Silicon Mac (M3+)** for development.
- **Apple Developer account** if targeting iOS / iPadOS on real devices. Free is fine for sideloaded builds; paid is required for App Store and the increased-memory entitlement.
- **A target device that meets the [hardware floor](/swift-sdk/overview#hardware-floor)** — under-spec devices are refused at runtime.

### New to Swift Package Manager?

Two on-ramps:

1. **Existing Xcode project** — skip to [Step 1](#1-add-the-package). You'll add `bitHumanKit` through Xcode's UI.
2. **Starting from scratch** — Xcode → **File → New → Project** → pick "App" (macOS / iOS / iPadOS) → name it → create. Then follow Step 1.

For a command-line tool or SPM-only project, edit `Package.swift` directly (Step 1 has the snippet).

## 1. Add the package

**In Xcode**: **File → Add Package Dependencies** → paste the URL → click "Add Package":

```
https://github.com/bithuman-product/bithuman-sdk-public.git
```

When Xcode asks which products to add, pick `bitHumanKit` and attach it to your app's target.

**In `Package.swift`**:

```swift
dependencies: [
    .package(
        url: "https://github.com/bithuman-product/bithuman-sdk-public.git",
        from: "0.8.1"
    )
],
targets: [
    .target(
        name: "MyApp",
        dependencies: [
            .product(name: "bitHumanKit", package: "bithuman-sdk-public")
        ]
    )
]
```

## 2. Permissions and entitlements

### All platforms — `Info.plist` privacy strings

```xml
<key>NSMicrophoneUsageDescription</key>
<string>Talk to your on-device assistant.</string>
<key>NSSpeechRecognitionUsageDescription</key>
<string>Recognise what you say so the assistant can respond.</string>
```

Without these, `chat.start()` fails silently — the OS denies the request and remembers the denial for the rest of the session.

### macOS only — sandboxed apps

If your Mac app is sandboxed (default for App Store distribution), also add to your `.entitlements`:

```xml
<key>com.apple.security.device.audio-input</key>
<true/>
```

Direct DMG / Homebrew distribution doesn't need the entitlement — only the `Info.plist` keys above.

### iOS only — memory entitlements (required)

<Warning>
Without these, iOS will terminate your app mid-conversation when memory exceeds the default ~3 GB ceiling (typically ~30 s into a live turn). Request approval **before** development starts — Apple typically takes 1–3 business days.
</Warning>

```xml
<key>com.apple.developer.kernel.increased-memory-limit</key>
<true/>
<key>com.apple.developer.kernel.extended-virtual-addressing</key>
<true/>
```

Request approval at developer.apple.com → **Account → Membership → Request Additional Capabilities**. The provisioning profile updates automatically once Apple replies.

## 3. Boot a voice chat (audio-only, unmetered)

```swift
import SwiftUI
import bitHumanKit

@main
struct MyApp: App {
    @StateObject private var lifecycle = MyLifecycle()

    var body: some Scene {
        WindowGroup {
            ContentView(lifecycle: lifecycle)
                .task { await lifecycle.start() }
        }
    }
}

@MainActor
final class MyLifecycle: ObservableObject {
    @Published var status = "booting…"
    private var chat: VoiceChat?

    func start() async {
        var config = VoiceChatConfig()
        config.localeIdentifier = "en-US"
        config.systemPrompt = "You are a helpful assistant. One sentence per turn."
        config.voice = .preset("Aiden")

        do {
            let chat = VoiceChat(config: config)
            try await chat.start()
            self.chat = chat
            status = "live — talk to me"
        } catch {
            status = "error: \(error.localizedDescription)"
        }
    }
}

struct ContentView: View {
    @ObservedObject var lifecycle: MyLifecycle
    var body: some View {
        Text(lifecycle.status).font(.title)
    }
}
```

Run it. Say "hello." The bot transcribes, thinks, replies via TTS through the speakers. That's the entire integration for an audio-only voice agent — no API key required, fully offline.

## 4. Add the lip-synced avatar (metered)

<Info>
Avatar mode charges 2 credits per active minute against your bitHuman account. Audio-only (Step 3) is unmetered. Get an API key at [www.bithuman.ai → Developer → API Keys](https://www.bithuman.ai/#developer) and set `VoiceChatConfig.apiKey` or export `BITHUMAN_API_KEY` before calling `chat.start()`.
</Info>

First launch downloads ~1.6 GB of avatar weights. Surface progress to the UI — silent hang is a reliable way to lose users.

```swift
import Foundation
import SwiftUI
import bitHumanKit

@MainActor
final class AvatarLifecycle: ObservableObject {
    @Published var phase: BootPhase = .idle
    @Published private(set) var coordinator: AvatarCoordinator?
    @Published private(set) var renderer: AvatarRendererView?
    private var chat: VoiceChat?
    private var pump: FramePump?

    enum BootPhase: Equatable {
        case idle
        case downloading(Double)   // 0…1
        case warming
        case live
        case error(String)
    }

    func start() async {
        do {
            phase = .downloading(0)
            let weights = try await ExpressionWeights.ensureAvailable { event in
                Task { @MainActor in
                    if case .downloading(let f, _, _, _, _) = event {
                        self.phase = .downloading(f)
                    }
                }
            }
            phase = .warming

            let agent = AgentCatalog.defaultAgent
            let portrait = AgentCatalog.thumbnailURL(for: agent)!

            var config = VoiceChatConfig()
            config.systemPrompt = agent.systemPrompt
            config.avatar = AvatarConfig(modelPath: weights, portraitPath: portrait)
            config.apiKey = ProcessInfo.processInfo.environment["BITHUMAN_API_KEY"]
            let chat = VoiceChat(config: config)
            try await chat.start()
            await chat.setVoicePreset(agent.voicePreset)

            guard let bh = chat.bithuman else {
                phase = .error("avatar engine failed to initialise")
                return
            }
            let coord = AvatarCoordinator(chat: chat)
            coord.bindToOrchestrator()
            coord.prewarmPortraitURL = portrait
            coord.currentAgentCode = agent.code

            let renderer = AvatarRendererView(
                frame: .zero, idleFrame: chat.initialIdleFrame, clipMode: .circle)
            let pump = FramePump(
                bithuman: bh, chat: chat, window: renderer, coordinator: coord)
            coord.framePump = pump
            chat.onBargeIn = { [weak pump] in pump?.buffer.flushSpeech() }

            // Hold strong refs so SwiftUI doesn't deinit them on re-render.
            self.chat = chat
            self.pump = pump
            self.renderer = renderer
            self.coordinator = coord
            self.phase = .live
        } catch {
            phase = .error(error.localizedDescription)
        }
    }
}
```

Host the renderer in your SwiftUI tree. The representables are explicit struct types because `NSViewRepresentable` / `UIViewRepresentable` are protocols — they don't take a closure. Returning the **same** renderer instance from both `makeXxxView` and `updateXxxView` is essential; SwiftUI may rebuild the parent many times per second, but the renderer must persist or the avatar disappears.

```swift
#if canImport(AppKit)
import AppKit
struct AvatarHost: NSViewRepresentable {
    let view: AvatarRendererView
    func makeNSView(context: Context) -> AvatarRendererView { view }
    func updateNSView(_ nsView: AvatarRendererView, context: Context) {}
}
#elseif canImport(UIKit)
import UIKit
struct AvatarHost: UIViewRepresentable {
    let view: AvatarRendererView
    func makeUIView(context: Context) -> AvatarRendererView { view }
    func updateUIView(_ uiView: AvatarRendererView, context: Context) {}
}
#endif

// In your SwiftUI tree:
if let renderer = lifecycle.renderer {
    AvatarHost(view: renderer)
        .frame(width: 280, height: 280)
        .clipShape(Circle())
}
```

## 5. Add the hardware gate (iOS)

```swift
@main
struct MyApp: App {
    var body: some Scene {
        WindowGroup {
            switch HardwareCheck.evaluate() {
            case .supported:
                ContentView()
            case .unsupported(let reason):
                UnsupportedDeviceView(reason: reason)
            }
        }
    }
}
```
