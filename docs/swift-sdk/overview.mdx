`bitHumanKit` is a Swift Package that drops a real-time, lip-synced avatar into your iOS, iPadOS, or macOS app. Audio in, video frames out at 25 FPS. Inference runs on the device's CPU, GPU, and Neural Engine.

A lightweight billing heartbeat (one request per minute) keeps the avatar pipeline metered; audio-only mode is unmetered.

## Install

```swift
.package(
    url: "https://github.com/bithuman-product/bithuman-sdk-public.git",
    from: "0.8.1"
)
```

```swift
.product(name: "bitHumanKit", package: "bithuman-sdk-public")
```

```swift
import bitHumanKit
```

The package ships a single binary XCFramework (macOS arm64 + iOS device arm64 + iOS simulator arm64). All dependencies are bundled — your app's `Package.swift` needs only this one entry.

## What you get

- **Real-time avatar** — 25 FPS at the model's native resolution, on-device.
- **Built-in audio I/O** — optional voice components with on-device speech recognition and synthesis on supported hardware.
- **SwiftUI ready** — frames are delivered as `CGImage`; drop into `Image(uiImage:)` / `Image(nsImage:)`.
- **Cross-platform Swift** — the same code targets Mac, iPad, and iPhone.

## Hardware floor

For **Expression** on-device:

| Platform | Minimum |
|---|---|
| **macOS** | M3+ Apple Silicon, macOS 26 |
| **iPadOS** | iPad Pro M4+, 16 GB unified memory, iPadOS 26 |
| **iOS (iPhone)** | Not supported yet — use Essence instead. |

For **Essence** on-device — supported across the full Apple lineup (Mac M3+, iPad Pro M4+, iPhone 17 Pro+). See [Essence on Swift](/swift-sdk/essence) for details.

The SDK gates this at runtime — under-spec devices see a polite refusal screen instead of a half-loaded engine.

## Get an API key

The avatar pipeline charges 2 credits per active minute for Expression and 1 credit per minute for Essence. Audio-only mode is unmetered.

1. Sign in at [www.bithuman.ai → Developer → API Keys](https://www.bithuman.ai/#developer).
2. Export `BITHUMAN_API_KEY` in your app's environment (or pass it through your secret store).
3. The SDK authenticates on first runtime creation — bad keys fail fast.
