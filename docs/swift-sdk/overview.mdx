`bitHumanKit` is a Swift Package that drops a real-time voice
agent — with optional lip-synced avatar — into your Mac, iPad,
or iPhone app. All inference runs on the device's GPU and
Neural Engine. The avatar engine is metered through a 1-request-per-minute
heartbeat to `api.bithuman.ai`; audio-only mode runs unmetered.

## Install

```swift
.package(url: "https://github.com/bithuman-product/bithuman-sdk",
         from: "1.17.0")
```

The package ships a single `Bithuman` product wrapping a 3-slice xcframework (macOS arm64 + iOS device arm64 + iOS simulator arm64). Static link of `libessence` — zero transitive SwiftPM dependencies. The legacy `bitHumanKit` package distributed via `bithuman-sdk-public` (`from: "0.8.1"`) continues to work for existing consumers; new integrations should target the unified `Bithuman` module above.

<Note>
The `BithumanRealtimeOpenAI` SwiftPM product was removed in v1.17.0. Its only consumer was the deleted Swift `bithuman-cli`; the canonical OpenAI Realtime path is now the Rust CLI's `bithuman voice --openai` and `bithuman avatar --openai` modes (`brew install bithuman`).
</Note>

## What you get

- **Voice agent** — on-device speech recognition (Apple's `SpeechAnalyzer`),
  on-device language model, on-device voice synthesis with optional
  voice cloning.
- **Lip-synced avatar** — 384×384 @ 25 FPS, eight bundled agent
  personas, drag-drop face swap.
- **SwiftUI components** — `AvatarRendererView`, `AvatarCoordinator`,
  pickers for agents/voices/prompts, splash with progress.
- **Cross-platform** — same Swift code runs on Mac, iPad, iPhone.

## Hardware floor

For **Expression** on-device (DiT pipeline):

| Platform | Minimum | Notes |
|---|---|---|
| **macOS** | M3+ Apple Silicon, macOS 26 (Tahoe) | int4 DiT @ 384², ~750 MB resident |
| **iPad** | iPad Pro M4+, 16 GB unified memory, iPadOS 26 | Stage Manager + Picture-in-Picture in [`expression/ipad/`](https://github.com/bithuman-product/bithuman-apps/tree/main/expression/ipad) |
| **iPhone** | Not supported in v1.17.0 | Expression DiT exceeds the per-app memory cap on a single conversation turn. iPhone today is **Essence-only** — see [Essence on Swift](/swift-sdk/essence). |

For **Essence** on-device (libessence streaming) — supported across the
full Apple lineup (Mac M3+, iPad Pro M4+, iPhone 17 Pro+). See
[Essence on Swift](/swift-sdk/essence) for details.

`HardwareCheck.evaluate()` gates this at runtime — under-spec
devices see a polite refusal screen instead of a half-loaded engine.

## Get an API key

The avatar pipeline charges 2 credits per active minute. Audio-only
mode is unmetered.

1. Sign in at [https://www.bithuman.ai](https://www.bithuman.ai) → Developer → API Keys.
2. Set `VoiceChatConfig.apiKey` or export `BITHUMAN_API_KEY`.
3. The SDK authenticates synchronously on `chat.start()` —
   bad keys fail fast.
