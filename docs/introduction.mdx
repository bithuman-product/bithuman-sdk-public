bitHuman runs real-time lip-synced avatars **on any device** — from a $35 Raspberry Pi to an iPhone to an NVIDIA data center GPU. **No cloud required.** All inference runs locally on the device: push audio in, get animated video out at 25 FPS with under 200ms latency. Whether you are building an edge kiosk, a native mobile app, or a cloud-scale web service, bitHuman runs where your users are.

<Note>
**v1.16.0 is streaming-first.** The new C ABI v6 splits the runtime into a cheap `pushAudio` append and a constant-time `pullFrame` pull, replacing the legacy "pass the entire accumulated buffer per tick" loop that grew O(n²) and missed the 40 ms render budget after ~10 s. Surfaced today in the [Swift](/swift-sdk/streaming) and [Kotlin](/kotlin-sdk/streaming) bindings; the Python wheel `bithuman` is still on the legacy 1.12.4 batched API.
</Note>

## Runs everywhere — on-device, no cloud required

bitHuman is the only avatar SDK that runs high-quality real-time lip-sync on-device across every major platform. Local inference means lower latency, full privacy, and no dependency on cloud availability.

| Platform | Models | Runtime | Min Hardware |
|---|---|---|---|
| **Linux x86_64** | Essence | Python SDK | Any CPU, 4 GB RAM |
| **Linux ARM64** | Essence | Python SDK | Raspberry Pi 4B+ |
| **macOS Intel** | Essence | Python SDK | Any Intel Mac |
| **macOS Apple Silicon** | Essence + Expression | Python + Swift SDK | M3+ for Expression |
| **Windows x86_64** | Essence | Python SDK | Any CPU, 4 GB RAM |
| **iPadOS** | Expression + Essence | Swift SDK | iPad Pro M4+ |
| **iOS** | Expression | Swift SDK | iPhone 16 Pro+ (A18 Pro) |
| **NVIDIA GPU (Docker)** | Expression | Docker container | 8 GB+ VRAM |
| **bitHuman Cloud** | Essence + Expression | REST API / LiveKit | API key only |

Every on-device surface performs local inference — audio processing, lip-sync model execution, and frame rendering all happen on the hardware in front of the user. The only network call is a lightweight billing heartbeat (1 request per minute). Audio-only mode on Apple Silicon works fully offline with zero network calls.

## Pick your path

bitHuman ships **two models** and **three runtime surfaces**. Your choice depends on what you are shipping (web, server, or native app) and which avatar style fits.

### Two models

| Model | What it is | Avatar source | Compute | Best for |
|---|---|---|---|---|
| **[Essence](/getting-started/models#essence)** | CPU-only pre-rendered animations with real-time lip-sync | `.imx` model file built from a photo or video on [bithuman.ai](https://www.bithuman.ai/#explore) | CPU (any laptop, Raspberry Pi, edge box) | Kiosks, 24/7 displays, full body + animal mode, custom gestures |
| **[Expression](/getting-started/models#expression)** | AI-generated facial animation from a single image | Any face image — no avatar generation step | NVIDIA GPU **or** Apple Silicon M3+ (on-device) | Dynamic conversations, custom faces, native consumer apps |

### Three runtime surfaces

| Surface | What it is | Models supported | Best for |
|---|---|---|---|
| **[bitHuman Cloud](/deployment/livekit-cloud-plugin)** | Managed runtime on `api.bithuman.ai`. Use via the LiveKit plugin (`pip install livekit-plugins-bithuman`) or the [REST API](/api-reference/overview) directly. | Essence, Expression | Web apps, fastest setup, no infra. |
| **[Self-hosted server](/deployment/avatar-sessions)** | Run on your own hardware. Python SDK (`pip install bithuman`) for Essence on CPU, or [Docker container](/deployment/self-hosted-gpu) for Expression on Linux + NVIDIA. | Essence (CPU), Expression (GPU) | On-prem privacy, edge devices, high volume. |
| **[On-device Apple Silicon](/swift-sdk/overview)** | `bitHumanKit` Swift Package + `bithuman-cli` Homebrew tool. All inference (STT, LLM, TTS, lip-sync) runs locally; only a 1-req/min billing heartbeat goes out. | Expression (M3+ Mac, M4+ iPad Pro, A18 Pro+ iPhone) | Native consumer / pro-sumer apps where everything must run locally. |

## 5-minute quickstart

Pick the surface that matches what you're building:

<CodeGroup>

```bash Cloud (Docker stack)

cp .env.example .env
# Edit .env: set BITHUMAN_API_SECRET, BITHUMAN_AGENT_ID, OPENAI_API_KEY

```

```python Self-hosted Python (Essence, CPU)
# pip install bithuman
import asyncio, os
from bithuman import AsyncBithuman
from bithuman.audio import load_audio, float32_to_int16

async def main():
    runtime = await AsyncBithuman.create(
        model_path="avatar.imx",
        api_secret=os.environ["BITHUMAN_API_SECRET"],
    )
    await runtime.start()

    pcm, sr = load_audio("speech.wav")
    pcm = float32_to_int16(pcm)
    chunk = sr // 25
    for i in range(0, len(pcm), chunk):
        await runtime.push_audio(pcm[i : i + chunk].tobytes(), sr)
    await runtime.flush()

    async for frame in runtime.run():
        if frame.has_image:
            image = frame.bgr_image      # numpy (H, W, 3) uint8
        if frame.end_of_speech:
            break
    await runtime.stop()

asyncio.run(main())
```

```bash Self-hosted GPU (Expression, NVIDIA)
docker run --gpus all -p 8089:8089 \
    -e BITHUMAN_API_SECRET=your_api_secret \

# Then point a LiveKit agent at http://localhost:8089/launch
# See /deployment/self-hosted-gpu for the full wiring.
```

## Core SDK API (Python)

| Method | Description |
|--------|-------------|
| `AsyncBithuman.create(model_path, api_secret)` | Initialize the avatar runtime |
| `runtime.start()` | Begin processing |
| `runtime.push_audio(data, sample_rate)` | Send audio for lip-sync |
| `runtime.flush()` | Signal end of audio input |
| `runtime.run()` | Async generator yielding video + audio frames |
| `runtime.get_frame_size()` | Returns `(width, height)` of output |

## REST API

| Endpoint | Method | Description |
|----------|--------|-------------|
| `/v1/validate` | POST | Verify API secret |
| `/v1/agent/generate` | POST | Generate new avatar agent |
| `/v1/agent/{code}` | GET/POST | Get or update agent |
| `/v1/agent/{code}/speak` | POST | Make avatar speak text |
| `/v1/agent/{code}/add-context` | POST | Inject silent knowledge |
| `/v1/files/upload` | POST | Upload image/video/audio |
| `/v1/dynamics/generate` | POST | Generate gesture animations |
| `/v2/credit-summaries` | GET | Check credit balance and plan details |
| `/v1/embed-tokens/request` | POST | Generate JWT token for website embedding |

bitHuman performs on-device local inference on every major platform. No cloud round-trip for avatar rendering — all lip-sync runs on the hardware where your app runs.

| Platform | Status | On-device models | Notes |
