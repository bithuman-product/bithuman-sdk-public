## 2. Install

```bash
pip install bithuman --upgrade
```

Python 3.10 – 3.13. Wheels for macOS arm64, Linux x86_64, Linux aarch64, and Windows x86_64.

<Note>
The wheel pulls in `opencv-python-headless` automatically. Don't install full `opencv-python` alongside — it conflicts with PyAV and causes FFmpeg warnings on macOS.
</Note>

## 3. Run your first avatar

You need a `.wav` file to drive the avatar. Each [example directory](https://github.com/bithuman-product/bithuman-sdk-public/tree/main/Examples) bundles a sample `speech.wav`, or generate your own with any TTS service.

### Option A — One-line CLI render

```bash
export BITHUMAN_API_SECRET=your_api_secret
bithuman generate avatar.imx --audio speech.wav --output demo.mp4
```

Open `demo.mp4`. That's the fastest path — no Python code at all.

### Option B — Python streaming

```python
import asyncio, os
from bithuman import AsyncBithuman
from bithuman.audio import float32_to_int16, load_audio

async def main():
    runtime = await AsyncBithuman.create(
        model_path="avatar.imx",
        api_secret=os.environ["BITHUMAN_API_SECRET"],
    )
    await runtime.start()

    pcm, sr = load_audio("speech.wav")
    pcm = float32_to_int16(pcm)
    chunk = sr // 100                            # 10 ms per chunk
    for i in range(0, len(pcm), chunk):
        await runtime.push_audio(
            pcm[i : i + chunk].tobytes(), sr, last_chunk=False,
        )
    await runtime.flush()

    try:
        async for frame in runtime.run():
            if frame.has_image:
                image = frame.bgr_image          # numpy (H, W, 3) uint8
                # Display it, encode it, push it to WebRTC — your choice.
            if frame.end_of_speech:
                break
    finally:
        await runtime.stop()

asyncio.run(main())
```

To see the frames live, add OpenCV display + speaker playback — that's exactly what [Examples/python/local-essence/quickstart.py](https://github.com/bithuman-product/bithuman-sdk-public/tree/main/Examples/python/local-essence) does.

## Key concepts

| Concept | What it is |
|---|---|
| **`AsyncBithuman`** | The runtime. One instance per avatar session. |
| **`push_audio(bytes, sr, last_chunk)`** | Feed 16-bit PCM. Avatar lip-syncs in real time. |
| **`flush()`** | Mark end of audio input. |
| **`run()`** | Async generator that yields frames at 25 FPS. |
| **`frame`** | Has `.bgr_image` (numpy), `.audio_chunk`, `.has_image`, `.end_of_speech`. |

## Troubleshooting

<AccordionGroup>
  <Accordion title="ModuleNotFoundError: No module named 'bithuman'">
    The SDK isn't installed. Run `pip install bithuman --upgrade` in the same Python environment you're using to run your script.
  </Accordion>

  <Accordion title="401 / authentication failed">
    1. Check you copied the full secret from [Developer → API Keys](https://www.bithuman.ai/#developer).
    2. Check `BITHUMAN_API_SECRET` is set in the shell that runs your script.
    3. Verify your account is active:
       ```bash
       curl -X POST https://api.bithuman.ai/v1/validate -H "api-secret: $BITHUMAN_API_SECRET"
       ```
  </Accordion>

  <Accordion title="Avatar shows but no lip movement">
    1. Make sure you're calling `push_audio()` with PCM bytes (not float32 — use `float32_to_int16()`).
    2. Call `flush()` after pushing all audio.
    3. Verify the sample rate matches your file (use `load_audio()`).
  </Accordion>

  <Accordion title="Slow first startup (30-60 seconds)">
    First load of an `.imx` file converts older format files in place and warms the runtime. Subsequent loads in the same process are under 2 seconds. In production, keep the runtime alive between sessions.
  </Accordion>

  <Accordion title="FileNotFoundError: avatar.imx not found">
    Use an absolute path, or download a model from [Explore](https://www.bithuman.ai/#explore).
  </Accordion>
</AccordionGroup>
