Drop the `ai.bithuman:sdk` AAR into an Android Studio project and compose your first lip-synced frame. Takes about 10 minutes.

## Prerequisites

- **Android Studio** with NDK 28.0.13004108, compile SDK 35.
- **A target device with `arm64-v8a`**, Android 10+ (API 29+). Other ABIs aren't currently supported.
- **Mac or Linux dev box.** No platform-specific build tooling beyond Android SDK + NDK.
- **A bitHuman API secret** — sign in at [Developer → API Keys](https://www.bithuman.ai/#developer).

## 1. Add the dependency

In your app module's `build.gradle.kts`:

```kotlin
android {
    defaultConfig {
        ndk { abiFilters += setOf("arm64-v8a") }
        minSdk = 29
    }
}

dependencies {
    implementation("ai.bithuman:sdk:1.17.1")
}
```

The AAR pulls from Maven Central. Ensure `mavenCentral()` is in your `settings.gradle.kts` repositories block (it's there by default in new projects).

## 2. Provide an avatar model and audio

Push a `.imx` model file and a driving audio clip to the device's external files directory:

```sh
adb push sample-avatar.imx /sdcard/Android/data/your.app.id/files/
adb push speech.wav        /sdcard/Android/data/your.app.id/files/
```

Or fetch them inside the app over HTTPS and cache locally. The file path can be passed directly to `Avatar.load` — no need to copy into app assets.

## 3. Set the API secret

In `Application.onCreate()` (or wherever your secret store loads):

```kotlin
class App : Application() {
    override fun onCreate() {
        super.onCreate()
        System.setProperty("BITHUMAN_API_SECRET", BuildConfig.BITHUMAN_API_SECRET)
    }
}
```

## 4. Compose a frame

The high-level facade — one model file, one runtime, audio-file-driven:

```kotlin
import ai.bithuman.sdk.Avatar

val modelPath = "${filesDir}/sample-avatar.imx"
val audioPath = "${filesDir}/speech.wav"

Avatar.load(modelPath).use { avatar ->
    avatar.composeFromFile(audioPath).forEach { frame ->
        // frame.bgr is width*height*3 packed BGR uint8
        // Hand to ImageView via a Bitmap, or to a SurfaceView.
    }
}
```

For a raw 16 kHz mono `FloatArray` PCM source (e.g. live mic):

```kotlin
val pcm: FloatArray = /* 16 kHz mono, length = N * avatar.samplesPerTick */
Avatar.load(modelPath).use { avatar ->
    avatar.composeAsBitmaps(pcm).forEach { bitmap ->
        imageView.setImageBitmap(bitmap)
    }
}
```

`composeAsBitmaps` returns Android `Bitmap` instances directly. `composeFromFile` accepts WAV / MP3 / M4A and decodes through `MediaExtractor`.

## 5. Multi-conversation hosting

For server-style scenarios hosting multiple concurrent conversations, share one model file across many runtimes to amortize the load:

```kotlin
import ai.bithuman.sdk.Fixture
import ai.bithuman.sdk.Runtime

Fixture(modelPath).use { fx ->
    val rtA = Runtime(fx)
    val rtB = Runtime(fx)
    // Each runtime is independent; don't share a single runtime
    // across threads without external synchronization.
}
```

One `Fixture` backs many `Runtime` instances. Memory cost grows much more slowly per added conversation than loading the model fresh each time.

## 6. Choose an execution provider (optional)

The default backend is CPU — predictable and identical across platforms. Override via the low-level API:

```kotlin
import ai.bithuman.sdk.ExecutionProvider
import ai.bithuman.sdk.Fixture

Fixture(modelPath, preferredEp = ExecutionProvider.CPU, intraOpThreads = 1).use { fx ->
    // ...
}
```

`NNAPI` and `QNN` are accepted but currently no-op to CPU. CPU is the right choice for this workload.

## Common pitfalls

| Symptom | Fix |
|---|---|
| `UnsatisfiedLinkError` on app launch | Your build variant didn't include `arm64-v8a` — check `abiFilters` in `defaultConfig`. |
| `BithumanException: AUTH_FAILED` | API secret is missing or invalid. Confirm `BITHUMAN_API_SECRET` is set before the first `Avatar.load` call. |
| First compose tick is slow (~400 ms) | First-run initialization. Subsequent ticks are fast. Pre-warm by composing one silent tick at startup if you need consistent latency from the first frame. |
| `composeAsBitmaps` returns empty | PCM length must be ≥ `avatar.samplesPerTick`. Each tick consumes 640 samples (40 ms at 16 kHz). |
